# Contributing to BillMe

Thanks for your interest in contributing! The goal of this document is to help you get started quickly.

How to contribute
- File an issue to discuss bugs or feature requests.
- Fork the repo and create a feature branch.
- Follow the commit message style: short summary on first line, optional details after a blank line.
- Open a pull request describing your changes and link any related issues.

Coding guidelines
- The project is Kotlin/Android. Follow existing project structure and Kotlin coding conventions.
- Keep changes small and focused; one logical change per PR.

Development workflow
1. Clone the repository
2. Create a branch: `git checkout -b feat/your-feature`
3. Implement your changes and run `./gradlew build`
4. Run tests: `./gradlew test`
5. Push your branch and open a pull request

Before submitting
- Ensure all tests pass
- Add or update documentation where relevant
- Link the PR to any issues it resolves

Thank you for helping improve BillMe!
